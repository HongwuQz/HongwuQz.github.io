Hexo+Github个人博客的搭建

2020-03-29 教程 Hexo搭建

准备工作

  • 安装Nodejs

  • 本地用来存放博客的文件夹

  • 安装Git for windows

  • 在Github建一个new repository

  • 配置SSH公钥

本机构建

  • 安装npm以及淘宝镜像
  • 安装Hexo插件
  • 在本机用Git部署Hexo博客文件夹

架构与运营

  • 本地预览
  • 清理缓存
  • 生成网页
  • 架构到Github

安装Nodejs和Git for windows

在任何正规的软降网站下载都是可以的，我用的是软件管家

你也完全可以到Nodejs和Git的官网下载，需要注意的是Nodejs需要下载LTS稳定版本，Git按照自己电脑操作系统的位数下载windows、Linux或Mac的64位或32位版本

安装过程直接一路Next就可以了，安装完node之后就需要配置安装npm插件了,打开Nodejs command prompt

确定nodejs安装和环境变量配置没有问题之后，我们最好安装淘宝的镜像。不然你会发现网速慢到匪夷所思

安装完成之后我们检测一下

registry已经显示正常了,那么我们现在有了足够的网速就可以开始安装Hexo插件了,使用镜像cnpm install -g hexo-cli下载更快，如果镜像安装失败也可以直接用npm install -g hexo，

安装完Hexo老规矩我们确认一下是否安装成功

hexo插件安装好之后我们就可以先把博客文件部署到本地电脑上了

在Github建一个New repository

还没有Github的小伙伴可以到官网注册一个，注册完成之后右上角有个“+”号，可以新建一个仓库(repository)做为我们个人博客的存储仓库

==注意==：

新建的repository有特定要求，一定要是你的githubID.github.io。也就意味着每个Github用户只能构建一个Hexo,如果想要进行分类需要用到branch。这里暂时不做介绍

我们假如张三的Github用户名为ZhangSan,那么他的仓库名就必须是ZhangSan.github.io我们点Create repository，记住HTTPS或SSH的地址

)

创建本地博客文件夹

建议找一个自己容易找到的地方，例如:D:/Program Files/HBlogs建立好文件夹之后，我们刚才安装的Git就派上用场了。右键HBlogs选择Git Bash here，建立Hexo博客默认文件夹

此时打开我们的HBlogs文件夹就会发现多了很多文件。但是这样还不够，我们需要用npm install hexo --save来将Hexo搭建到我们设置的默认文件夹中

其中WARN不会产生影响，可以忽视，现在Hexo目前已经完成了初步搭建在了本地电脑上，我们可以用hexo g来生成网页。

现在我们用hexo s在本地端口生成预览,。

我们在网页直接输入localhost:4000就可以登陆查看预览了，如果登陆不进去可能是你的4000端口被占用了hexo s -p 5000换个端口即可

我们现在可以考虑把这个网页上传到github了，打开_config.yml，==注意==：_config.yml有两个。我们用编译器打开HBlog（记事本足够）目录下的这个文件，挪到最下方,输入以下内容

注意这三行每个冒号后面一定要加空格 ！加空格 ！加空格 ！ 不然会崩溃！！

repo那一栏填上面新建new repository时的网址，HTTPS和SSH都可以。现在保存退出，用hexo d测试一下

显示找不到托管对象git，原因是还需要安装一个插件npm install hexo-deployer-git --save

这里面有一个WARN是提醒Mac系统安装fsevents的，windows系统可以无视掉,那么我们再来测试一下hexo d

期间会弹出对话框要你输入你的Github和密码，对应输入即可。之后直接登陆到https://ZhangSan.github.io即可看到你的博客啦，如果连不上可以稍等并且刷新一下，我就直接拿我的博客做展示了。
